# 수동 QA 체크리스트

공개 배포 전 Streamlit Cloud 앱에서 아래 항목을 순서대로 확인한다.

## 계정과 온보딩

- [ ] 신규 회원가입이 성공한다.
- [ ] 이메일 확인이 켜져 있으면 인증 후 로그인할 수 있다.
- [ ] `로그인 유지`를 체크하고 로그인한 뒤 새로고침하면 같은 브라우저에서 자동 로그인된다.
- [ ] 로그아웃하면 저장된 로그인 유지 세션이 삭제되고 다시 로그인 화면이 보인다.
- [ ] Streamlit Secrets에 `AUTH_SESSION_SECRET`이 없으면 로그인 유지 체크박스가 비활성화된다.
- [ ] 신규 로그인 직후 `처음 시작하기` 온보딩 화면이 보인다.
- [ ] `샘플 포트폴리오로 둘러보기`를 누르면 샘플 보유종목, 현금, 목표 비중이 표시된다.
- [ ] 샘플 모드에서는 저장 상태가 `샘플 모드 - 저장 안 됨`으로 표시된다.
- [ ] `샘플 데이터 삭제`를 누르면 샘플 데이터가 사라진다.

## 현금 원장

- [ ] KRW 입금을 저장하면 원화 현금이 증가한다.
- [ ] USD 입금을 저장하면 달러 현금이 증가한다.
- [ ] 배당 입력을 저장하면 해당 통화 현금이 증가한다.
- [ ] 출금 입력을 저장하면 해당 통화 현금이 감소한다.
- [ ] 수동 조정은 현재 원장 잔고와 목표 잔고의 차이만큼 원장 row를 추가한다.
- [ ] 현금 원장 표에서 날짜, 구분, 통화, 금액, 메모를 확인할 수 있다.

## 거래 입력

- [ ] 국내 주식 매수 저장 후 KRW 현금이 체결금액, 수수료, 세금만큼 감소한다.
- [ ] 미국 주식 매수 저장 후 USD 현금이 체결금액, 수수료, 세금만큼 감소한다.
- [ ] 주식 매도 저장 후 해당 통화 현금이 증가한다.
- [ ] 보유 수량 초과 매도는 저장 전에 차단된다.
- [ ] 거래 입력의 가격 라벨은 `체결단가`로 표시된다.
- [ ] 보유현황의 평균 매입가격 라벨은 `평균단가` 또는 `평균 매입단가`로 표시된다.

## 가격과 환율

- [ ] `가격·환율 갱신`을 누르면 모든 보유 종목과 USD/KRW 환율을 다시 조회한다.
- [ ] 성공/캐시/실패/미조회 수가 갱신 요약에 표시된다.
- [ ] 보유현황 상세 열에서 가격 기준일, 조회 시각, 출처, 상태를 확인할 수 있다.
- [ ] 환율 조회 실패 시 기존 환율 유지 메시지가 명확히 표시된다.
- [ ] 가격·환율 갱신 중 외부 가격 API가 실패해도 앱 shell, 상단 탭, 현재 입력값이 유지된다.

## 총괄현황과 매매일지

- [ ] 총괄현황의 기존 히트맵이 표시된다.
- [ ] 총괄현황의 기존 보유 종목 총괄 표가 표시된다.
- [ ] 총괄현황에 투자자산과 현금이 별도 요약으로 표시된다.
- [ ] 매매일지 탭에서 매입/매도 거래가 날짜순으로 표시된다.
- [ ] 연결된 매입/매도 거래와 cash_ledger 정산 row가 중복 이벤트로 보이지 않는다.
- [ ] 독립 입금/출금, 환전, 배당/이자 원장이 매매일지에 표시된다.
- [ ] 수동 메모를 작성하고 새로고침/재로그인 후 복원되는지 확인한다.

## 성과, 리밸런싱, 리스크

- [ ] 성과분석에서 총손익, 실현손익, 미실현손익, 배당/이자, 수수료/세금, 환율효과를 확인할 수 있다.
- [ ] 목표 비중을 저장할 수 있다.
- [ ] 목표 비중 합계가 100%가 아니면 경고가 표시된다.
- [ ] 리밸런싱 결과에서 현재 비중, 목표 비중, 차이 금액, 조정 수량을 확인할 수 있다.
- [ ] MDD와 현재 고점 대비 하락률이 표시된다.
- [ ] 벤치마크 선택 후 Beta가 표시되거나 데이터 부족 안내가 표시된다.

## CSV 백업과 복원

- [ ] 거래 CSV 템플릿을 다운로드할 수 있다.
- [ ] 현금 원장 CSV 템플릿을 다운로드할 수 있다.
- [ ] 거래 CSV 업로드 후 저장 전 미리보기와 validation 결과가 보인다.
- [ ] 현금 원장 CSV 업로드 후 저장 전 미리보기와 validation 결과가 보인다.
- [ ] 잘못된 행은 저장되지 않고 행 번호와 오류 이유가 표시된다.
- [ ] 중복 거래 또는 중복 원장은 저장되지 않는다.
- [ ] 전체 데이터 JSON 내보내기가 가능하다.
- [ ] 거래 CSV, 현금 원장 CSV, 목표 비중 CSV 내보내기가 가능하다.

## 화면과 테마

- [ ] 모바일 390px, 태블릿 768px, 데스크톱 1024px 이상에서 주요 탭이 깨지지 않는다.
- [ ] 라이트 모드 총괄현황의 배경, 카드, 히트맵, 보유 종목 표 색상이 서로 구분된다.
- [ ] 다크 모드 총괄현황의 배경, 카드, 히트맵, 보유 종목 표 색상이 서로 구분된다.
- [ ] 라이트/다크에서 수익은 rose/red 계열, 손실은 blue 계열로 일관되게 표시된다.
- [ ] 라이트/다크에서 저장 성공, 경고, 오류, 안내 박스가 손익 색상과 혼동되지 않는다.
- [ ] 라이트/다크에서 리밸런싱 data editor 텍스트와 헤더가 읽힌다.
- [ ] 라이트/다크에서 매매일지 timeline 카드의 날짜, 구분, 금액, 원본 상세가 읽힌다.
- [ ] 라이트/다크에서 차트 축, legend, tooltip 텍스트가 읽힌다.
- [ ] 모바일 390px에서 손익 색상과 카드 텍스트 대비가 충분하다.
- [ ] 색상 전환 후 widget key warning이 발생하지 않는다.
- [ ] 색상 전환 후 active tab과 active subtab이 유지된다.
- [ ] 모바일 화면에서 총괄현황, 세부내역, 사용자입력, 자산추이, 매매일지, 리밸런싱 탭이 탭 형태로 보인다.
- [ ] 사용자입력 하위 탭 `보유 현황`, `현금·입출금·환율`, `거래 입력`, `CSV`가 한 줄에 표시된다.
- [ ] 다크 모드에서 카드, 표, 차트 텍스트가 읽힌다.
- [ ] 라이트 모드에서 카드, 표, 차트 텍스트가 읽힌다.
- [ ] 총괄현황/세부내역/성과분석/리밸런싱을 라이트/다크 각각 스크린샷으로 남긴다.
- [ ] 라이트/다크 전환 시 Streamlit widget key warning이 표시되지 않는다.
- [ ] 라이트/다크 전환 후 선택한 상위 탭과 하위 탭, 입력 중인 현금/거래 값이 유지된다.
- [ ] 빈 포트폴리오에서는 0원 차트 대신 다음 입력 안내가 표시된다.
- [ ] 값이 모두 0인 성과/리스크/자산추이 차트는 빈 상태 안내로 대체된다.
- [ ] 일부 탭에서 오류가 발생해도 전체 앱이 흰 화면이 되지 않고 오류 ID가 있는 안내가 표시된다.
- [ ] 가격/환율 갱신 실패 후에도 마지막 정상값 또는 명확한 실패 상태가 표시된다.
- [ ] 빈 계정에서 상단 `가격·환율 갱신`을 눌러도 외부 가격/환율 조회가 시작되지 않고 조회 대상 없음 안내가 표시된다.
- [ ] 가격·환율 갱신 중 버튼이 중복 실행되지 않고 조회 중 상태가 표시된다.
- [ ] 외부 가격 API를 느리게 하거나 차단해도 갱신이 무기한 멈추지 않고 남은 종목은 마지막 정상 가격을 유지한다.
- [ ] 1분 자동갱신 중에는 종목별 분봉 조회가 반복되지 않고 현재가와 환율만 갱신된다.
- [ ] 자산추이에서 실제 기록, 성과분석, 리스크분석, 과거 보유현황 재구성 중 선택한 화면만 렌더링된다.
- [ ] CSV 화면에서 가져오기와 내보내기 중 선택한 화면만 렌더링된다.
- [ ] 자산추이 기록 저장이 실패해도 포트폴리오 저장과 현재 화면은 유지된다.
- [ ] 로그인 후 탭을 20회 이상 이동해도 최초 로딩 화면으로 돌아가거나 흰 화면이 되지 않는다.
- [ ] 앱을 1시간 이상 열어 둔 뒤 탭 이동, 자동 저장, 수동 가격 갱신이 정상 동작한다.
- [ ] 서로 다른 브라우저에서 두 계정으로 동시에 로그인해도 계정 정보와 저장 데이터가 섞이지 않는다.
- [ ] 두 브라우저에서 동시에 가격 갱신을 시작해도 KIS 토큰 발급 오류로 앱이 멈추지 않는다.
- [ ] `가격·환율 갱신`, `입출금 저장`, `거래 반영`, `목표 비중 저장`, `샘플 데이터 삭제`를 빠르게 연속 클릭해도 흰 화면이 되지 않고 중복 반영 또는 멈춤 상태가 발생하지 않는다.
- [ ] 작업 직후 다른 버튼을 너무 빠르게 누르면 짧은 안내 메시지가 표시되고, 잠시 뒤 다시 정상 조작할 수 있다.
- [ ] 리밸런싱 목표 비중을 저장한 뒤 새로고침/재로그인해도 값이 복원된다.
- [ ] `target_allocations` 테이블이 있는 환경에서는 목표 비중이 테이블 기준으로 복원되고, 테이블 접근 실패 환경에서는 기존 포트폴리오 payload 기준으로 복원된다.
- [ ] 기존 분류표에 없는 국내 또는 미국 종목을 추가하면 데스크톱 히트맵에서 자동 섹터 분류 또는 `기타`가 표시된다.
- [ ] 신규 종목의 섹터 분류 후 새로고침/재로그인해도 같은 분류가 복원된다.
- [ ] 메타데이터 조회를 차단해도 앱이 멈추지 않고 해당 종목이 `기타`에 표시된다.

## UI Polish / Visual Regression

- [ ] 총괄현황에서 treemap label이 겹치거나 잘리지 않는다.
- [ ] 총괄현황 보유종목 표의 기본 컬럼이 읽기 쉽다.
- [ ] 세부내역 도넛/통화노출/기여도 차트 라벨이 잘리지 않는다.
- [ ] 사용자입력 보유현황 필터 row가 정렬되어 있다.
- [ ] 사용자입력 거래 입력 form의 input과 버튼 baseline이 맞는다.
- [ ] 사용자입력 현금 form의 input과 버튼 baseline이 맞는다.
- [ ] CSV 업로드 영역의 버튼과 uploader 크기가 일관된다.
- [ ] 자산추이 실제기록 차트 x축이 과밀하지 않다.
- [ ] 성과분석 카드 높이가 일관된다.
- [ ] 성과분석 차트 라벨이 잘리지 않는다.
- [ ] 리스크분석 drawdown 축과 legend가 잘리지 않는다.
- [ ] 과거 보유현황 재구성 화면의 버튼 width가 일관된다.
- [ ] 매매일지 summary card와 filter chip 크기가 일관된다.
- [ ] 매매일지 event card 간격이 과하지 않다.
- [ ] 리밸런싱 옵션 버튼 크기가 일관된다.
- [ ] 리밸런싱 결과표 기본 컬럼이 읽기 쉽다.
- [ ] 라이트/다크 전환 후 layout이 깨지지 않는다.
- [ ] 390px mobile에서 가로 overflow가 없다.
- [ ] 768px tablet에서 chart/card가 겹치지 않는다.
- [ ] 1440px desktop에서 본문이 중앙 정렬되어 있다.
- [ ] 운영자 화면에서 Manage app overlay가 하단 버튼/표/차트를 가리지 않는다.

## RLS 사용자 분리

- [ ] 사용자 A의 데이터가 사용자 B 계정에서 보이지 않는다.
- [ ] 사용자 A의 `cash_ledger`를 사용자 B가 조회할 수 없다.
- [ ] 사용자 A의 `target_allocations`를 사용자 B가 수정할 수 없다.
- [ ] 로그아웃 또는 anon role에서 사용자 데이터 테이블을 조회할 수 없다.

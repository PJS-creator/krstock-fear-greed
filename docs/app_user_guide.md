# 포트폴리오 대시보드 소개 및 사용 설명서

## 앱 소개

포트폴리오 대시보드는 국내 주식, 미국 주식, 원화 현금, 달러 현금을 한 화면에서 관리하고 KRW 기준 총자산을 확인하는 개인 투자 관리 앱입니다.

사용자는 회원가입 후 본인 계정으로 로그인하고, 매입/매도 거래와 현금 정보를 입력하면 앱이 현재 보유현황, 총자산, 평가이익, 일간 변동, 자산 비중, 통화 노출, 자산추이를 자동으로 계산합니다.

공개 배포 버전은 사용자별 계정을 기준으로 하나의 기본 포트폴리오를 자동 저장하고 자동 불러옵니다. 별도의 포트폴리오 저장/불러오기 버튼을 누르지 않아도 같은 계정으로 다시 로그인하면 이전 입력값을 이어서 확인할 수 있습니다.

## 주요 기능

- 이메일/비밀번호 기반 회원가입 및 로그인
- 사용자별 포트폴리오 자동 저장 및 자동 불러오기
- 국내 주식, 미국 주식, 원화 현금, 달러 현금 통합 관리
- 표준 거래 입력 Form 기반 매입/매도 기록
- 현금 원장 기반 입금, 출금, 배당, 이자, 수수료, 세금 관리
- 매입/매도 거래 입력 기반 현재 보유현황과 현금 잔고 자동 계산
- USD/KRW 환율 입력 및 갱신
- 보유 종목 가격과 환율 일괄 갱신
- 총자산, 주식 평가금액, 현금, 평가이익, 수익률, 오늘 손익 확인
- 자산 비중, 통화 노출, 오늘 변동 기여도, 자산 진단 확인
- 거래, 현금 원장, 환전, 배당/이자, 수동 메모를 모아 보는 매매일지
- 실현손익, 미실현손익, 배당/이자, 수수료/세금, 환율효과 분해
- 목표 비중 설정 및 리밸런싱 계산
- MDD와 Beta 기반 리스크 분석
- 실제 저장 기록 기반 자산추이 확인
- 과거 보유현황 스냅샷 기반 자산 재구성
- 거래 CSV와 현금 원장 CSV 가져오기/내보내기
- 전체 데이터 JSON 내보내기
- 라이트 모드와 다크 모드 지원
- 모바일 화면 최적화

## 색상과 테마

앱은 라이트 모드와 다크 모드를 지원합니다. 오른쪽 위의 `다크` / `라이트` 버튼으로 전환할 수 있으며, 전환해도 선택한 탭과 입력 중인 값은 유지됩니다.

손익 색상은 한국 투자자에게 익숙한 기준을 따릅니다.

- 수익, 상승, 평가이익, 실현이익: rose/red 계열
- 손실, 하락, 평가손실, 실현손실: blue 계열
- 0, 미산정, 데이터 부족: neutral 색상
- 저장 성공/정상: success 색상
- 경고/데이터 부족/추정: warning 색상
- 실패/오류: danger 색상
- 일반 안내: info 색상

색상은 보조 정보이며, 앱은 `+`/`-` 기호와 상태 텍스트를 함께 표시합니다.

## 첫 사용 순서

1. 앱에 접속합니다.
2. **회원가입** 탭에서 이메일과 비밀번호로 계정을 만듭니다.
3. Supabase Auth 이메일 확인이 켜져 있으면 이메일 인증을 완료합니다.
4. **로그인** 탭에서 같은 이메일과 비밀번호로 로그인합니다.
   - 개인 기기라면 **로그인 유지**를 체크할 수 있습니다. 이 경우 같은 브라우저에서 최대 30일 동안 자동 로그인을 시도합니다.
   - 공용 PC나 다른 사람과 함께 쓰는 기기에서는 체크하지 마세요. 로그아웃하면 저장된 로그인 유지 세션도 삭제됩니다.
5. 첫 로그인 후 보이는 **처음 시작하기** 화면에서 시작 방식을 선택합니다.
6. 샘플로 둘러보거나, 현재 보유종목만 빠르게 입력하거나, 거래/현금 CSV를 업로드합니다.
7. 실제 데이터를 입력할 때는 먼저 **사용자입력 > 현금·입출금·환율**에서 KRW 또는 USD 입금을 기록합니다.
8. **사용자입력 > 거래 입력**에서 매입/매도 거래를 입력합니다.
9. 상단의 **가격·환율 갱신**을 눌러 현재 가격과 환율을 갱신합니다.
10. **총괄현황**, **세부내역**, **자산추이**, **매매일지**, **리밸런싱**에서 전체 자산 상태와 기록을 확인합니다.
11. 이후에는 같은 계정으로 로그인하면 `main` 포트폴리오가 자동으로 불러와집니다.

거래와 현금 데이터가 아직 없거나 모든 값이 0원이면 일부 차트는 표시되지 않고 다음 입력 안내가 표시됩니다. 먼저 현금 입금 또는 매입 거래를 입력한 뒤 가격·환율을 갱신하면 총괄현황과 차트가 채워집니다.

버튼을 누른 직후에는 화면 갱신과 자동 저장이 함께 실행될 수 있습니다. 같은 버튼이나 다른 저장 버튼을 너무 빠르게 연속으로 누르면 앱이 중복 처리를 막기 위해 잠시 기다리라는 안내를 표시할 수 있습니다. 안내가 보이면 화면 갱신이 끝난 뒤 다시 눌러 주세요.

## 온보딩

새 계정으로 로그인했고 아직 거래, 현금 원장, 보유종목 데이터가 없으면 **처음 시작하기** 화면이 표시됩니다.

선택지는 다음 3개입니다.

- **샘플 포트폴리오로 둘러보기**: 앱 기능을 빠르게 확인하는 예시 데이터입니다. 실제 투자 기록과 구분되며 자동 저장하지 않습니다. 필요 없으면 **샘플 데이터 삭제**를 누릅니다.
- **현재 보유종목만 빠르게 입력**: 종목명 또는 티커, 수량, 선택 평균단가만 입력해 현재 보유현황을 만들 수 있습니다.
- **거래/현금 CSV 업로드**: 거래내역과 현금 원장을 CSV로 가져옵니다. 저장 전 미리보기와 validation 결과를 먼저 확인합니다.

## 화면 구성

### 총괄현황

총괄현황은 현재 포트폴리오의 핵심 상태를 한눈에 보는 화면입니다.

확인할 수 있는 항목은 다음과 같습니다.

- 총자산
- 주식 평가금액
- 현금
- 평가이익
- 주식 수익률
- 오늘 손익
- 전일대비 수익률
- 투자원금
- 적용 환율
- 종목별 자산 비중
- 종목별 전일 대비 성과
- 보유 종목 요약

총자산은 주식 평가금액과 현금을 합산한 KRW 기준 금액입니다. 미국 주식과 달러 현금은 입력된 USD/KRW 환율로 원화 환산됩니다.

총괄현황의 히트맵과 보유 종목 표는 기존처럼 유지됩니다. 그 아래의 **투자자산 / 현금** 분리 요약은 총자산 중 주식 평가금액과 현금이 각각 얼마인지 따로 확인하기 위한 보조 영역입니다.

### 세부내역

세부내역은 포트폴리오를 더 자세히 분석하는 화면입니다.

확인할 수 있는 항목은 다음과 같습니다.

- 총자산, 오늘 변동, 총현금, USD 노출도
- 현금 포함 자산 비중 도넛 차트
- KRW/USD 통화 노출
- 오늘 변동 기여도
- 자산 진단

도넛 차트는 현금을 포함한 총자산 기준입니다. 통화 노출은 국내 자산과 원화 현금은 KRW, 미국 주식과 달러 현금은 USD로 분류해 KRW 환산 금액 기준으로 계산합니다.

### 매매일지

매매일지는 사용자의 투자 활동을 날짜순으로 복기하는 화면입니다.

표시되는 항목은 다음과 같습니다.

- 매입/매도 거래
- 입금/출금과 수동 현금 조정
- 환전
- 배당/이자
- 사용자가 직접 작성한 수동 메모

매입/매도 거래에 연결된 현금 정산 원장은 별도 이벤트로 중복 표시하지 않고, 해당 거래 이벤트의 현금 영향으로 함께 보여줍니다. 수동 메모는 날짜, 제목, 내용, 종목 연결, 태그를 입력해 저장할 수 있으며 기존 포트폴리오 자동 저장 payload에 함께 보존됩니다.

### 사용자입력

사용자입력은 포트폴리오 데이터를 입력하고 수정하는 화면입니다.

하위 항목은 다음과 같습니다.

- **보유 현황**: 현재 계산된 보유 종목과 평가액을 확인합니다.
- **현금·입출금·환율**: 입금, 출금, 배당, 이자, 환전, 수동 조정을 현금 원장으로 기록합니다.
- **거래 입력**: 매입/매도 거래를 입력해 보유현황을 계산합니다.
- **CSV**: 거래 CSV와 현금 원장 CSV를 가져오거나 데이터를 내보냅니다.

#### 거래 입력 방법

**표준 거래 입력** Form에서 아래 값을 입력합니다.

- `거래 구분`: 매입 또는 매도
- `시장`: KR, US 또는 자동감지
- `주식명 또는 티커`: 삼성전자, SK하이닉스, 005930, QURE, GOOGL 같은 종목명 또는 티커
- `체결단가`: 실제 거래된 1주당 가격입니다. 국내 주식은 원화, 미국 주식은 달러 기준입니다.
- `수량`: 거래 수량입니다.
- `거래통화`: 국내 주식은 기본 KRW, 미국 주식은 기본 USD입니다.
- `수수료`, `세금`: 0 이상으로 입력합니다.
- `거래일`: 달력에서 선택합니다.
- `메모`: 선택 입력입니다.

`체결단가`는 거래 입력 시점의 가격이고, 보유현황에 표시되는 `평균단가` 또는 `평균 매입단가`는 여러 매입/매도를 반영해 계산된 결과입니다.

거래를 저장하면 앱은 보유수량과 평균단가를 다시 계산합니다. 매수 거래는 해당 통화 현금을 감소시키고, 매도 거래는 해당 통화 현금을 증가시킵니다. 수수료와 세금도 현금 흐름에 반영됩니다. 보유수량보다 많은 매도 거래는 저장 전에 차단됩니다.

기존 빠른 입력, 붙여넣기 입력, CSV 업로드는 **고급 입력** 영역에 남아 있습니다.

#### 현금·입출금·환율

현금 잔고는 직접 입력한 숫자가 아니라 현금 원장의 합계로 계산됩니다.

주요 입력 방식은 다음과 같습니다.

- **입금**: 현금 증가입니다. 예: KRW 10,000,000 입금, USD 2,000 입금
- **출금**: 현금 감소입니다. 출금은 음수 원장으로 저장됩니다.
- **배당/이자**: 해당 통화 현금 증가로 저장됩니다.
- **수동 조정**: 목표 잔고와 현재 원장 잔고의 차이만큼 조정 원장을 추가합니다.
- **환전**: `fx_conversion_out`, `fx_conversion_in` 원장 2개를 만들어 From 통화 감소와 To 통화 증가를 기록합니다.

달러 현금과 미국 주식은 USD/KRW 환율을 적용해 KRW 기준 총자산에 반영됩니다. **USD/KRW 환율 갱신**을 누르면 앱이 무료 환율 데이터 소스를 조회합니다. 실패하면 기존 환율을 유지하고 실패 상태를 표시합니다.

#### CSV 가져오기와 내보내기

**사용자입력 > CSV**에서 백업과 복원을 할 수 있습니다.

- 거래 CSV 템플릿 다운로드
- 현금 원장 CSV 템플릿 다운로드
- 거래 CSV 업로드와 저장 전 미리보기
- 현금 원장 CSV 업로드와 저장 전 미리보기
- 전체 데이터 JSON 내보내기
- 거래 CSV, 현금 원장 CSV, 목표 비중 CSV 내보내기

업로드 시 오류 또는 중복으로 표시된 행은 저장하지 않습니다. 중복 거래는 `external_id`가 있으면 그 값을 우선 사용하고, 없으면 날짜, 종목, 구분, 수량, 가격, 메모 조합으로 감지합니다.

### 자산추이

자산추이는 시간에 따른 자산 변화를 확인하는 화면입니다.

하위 항목은 다음과 같습니다.

- **실제 기록**: 앱에 저장된 실제 포트폴리오 스냅샷 기반 총자산 변화입니다.
- **과거 보유현황 재구성**: 날짜별 보유현황과 현금/환율 스냅샷을 입력해 과거 자산 흐름을 재구성합니다.

#### 실제 기록

실제 기록은 저장된 포트폴리오 스냅샷을 기준으로 총자산 변화를 보여줍니다. 기록은 앱을 사용하고 저장이 누적된 시점부터 쌓입니다.

단순 화면 새로고침만으로 과거 기록이 새로 생성되지는 않습니다. 가격/환율 갱신과 저장이 이루어진 시점의 기록이 쌓이면 자산추이 차트에서 확인할 수 있습니다.

#### 과거 보유현황 재구성

과거 보유현황 재구성은 과거 특정 날짜의 전체 보유현황을 입력해 과거 총자산을 다시 계산하는 기능입니다.

이 기능은 거래내역 수익률 계산기가 아닙니다. 각 날짜의 입력값은 그 시점부터 다음 입력 날짜 전까지 유지되는 전체 보유현황 스냅샷입니다.

예를 들어 2026-04-13에 삼성전자 200주, GOOGL 22주를 입력하고 이후 수량 변화가 없으면, 앱은 2026-04-13부터 현재까지 같은 수량을 보유한 것으로 보고 날짜별 평가액을 재구성합니다.

### 성과분석

성과분석은 단순 평가손익을 더 세분화해 보여줍니다.

확인할 수 있는 항목은 다음과 같습니다.

- 총손익
- 실현손익
- 미실현손익
- 배당/이자 수익
- 수수료/세금 합계
- 환율효과
- 순입금액
- 입출금 제외 성과

수수료와 세금은 사용자가 입력한 값 기준의 집계입니다. 실제 세무 계산이나 세무 자문이 아닙니다.

### 리밸런싱

리밸런싱은 사용자가 직접 입력한 목표 비중과 현재 포트폴리오의 차이를 계산합니다.

사용 순서는 다음과 같습니다.

1. **리밸런싱** 탭으로 이동합니다.
2. 현재 보유 종목, KRW 현금, USD 현금 행을 확인합니다.
3. 각 자산의 목표 비중을 입력합니다.
4. 목표 비중 합계가 100%에 가까운지 확인합니다.
5. 결과 표에서 현재 평가액, 목표 평가액, 차이 금액, 조정 수량을 확인합니다.

이 기능은 `매수 추천` 또는 `매도 추천`이 아닙니다. 사용자가 입력한 목표 비중 대비 필요한 조정량을 계산하는 도구입니다.

아직 보유종목이나 현금이 없으면 리밸런싱 계산 대신 데이터 입력 안내가 표시됩니다. 목표 비중은 Supabase `target_allocations` 테이블이 사용 가능하면 그곳에 우선 저장되고, 테이블을 사용할 수 없는 환경에서는 기존 포트폴리오 자동 저장 payload에 함께 저장됩니다. 같은 계정으로 다시 로그인하면 저장된 목표 비중이 복원됩니다.

### 리스크 분석

리스크 분석에서는 MDD와 Beta를 확인할 수 있습니다.

- **MDD**: 관측 기간 동안 고점 대비 가장 크게 하락한 비율입니다.
- **현재 고점 대비 하락률**: 현재 총자산이 최근 고점 대비 얼마나 내려와 있는지 보여줍니다.
- **Beta**: 선택한 벤치마크 대비 포트폴리오 수익률의 민감도입니다.

MDD와 Beta는 과거 데이터 기반 지표이며 미래 손실이나 수익을 보장하거나 예측하지 않습니다. 데이터가 부족하면 계산하지 않고 데이터 부족 안내를 표시합니다.

## 가격·환율 갱신

상단의 **가격·환율 갱신** 버튼은 보유 중인 종목 가격과 USD/KRW 환율을 다시 조회합니다.

현재 사용하는 무료 데이터 소스는 다음과 같습니다.

- 미국 주식: 한국투자증권 Open API가 설정되어 있으면 우선 사용하고, 실패하면 yfinance 기반 최근 제공 가격을 사용합니다.
- 국내 주식: 한국투자증권 Open API가 설정되어 있으면 우선 사용하고, 실패하면 FinanceDataReader 기반 최근 제공 가격 또는 최근 종가를 사용합니다.
- USD/KRW 환율: Yahoo chart 및 fallback 환율 데이터

무료 데이터 소스는 실시간 시세를 보장하지 않습니다. 데이터 지연, 일시 차단, 특정 종목 누락, 환율 조회 실패가 발생할 수 있습니다. 조회 실패 시 마지막 정상 가격이 있으면 그 가격을 유지하고, 정상 가격도 없으면 평가액을 계산하지 않습니다.

각 가격과 환율에는 가능한 범위에서 기준일, 조회 시각, 출처, 상태가 함께 표시됩니다. `이전저장값사용`, `조회실패`, `환율실패_기존값유지` 같은 상태가 보이면 최신 조회에 실패했거나 마지막 정상 값을 사용하고 있다는 의미입니다.

PC 화면의 사이드바 **데이터 정보**에서는 현재 KIS 설정 상태, 최근 주식 조회 출처, 종목별 최근 조회 API, 환율 출처를 확인할 수 있습니다. 예를 들어 한국투자증권 Open API로 조회된 종목은 `한국투자 Open API`로 표시됩니다.

상단의 **가격·환율 갱신** 버튼 아래에 있는 **1분 자동갱신**을 켜면 앱이 열린 동안 60초마다 가격과 환율 갱신을 자동 시도합니다. 모바일 앱/WebView에서도 같은 상단 토글로 켜고 끌 수 있습니다. 자동갱신은 서버가 항상 백그라운드에서 돌면서 조회하는 기능이 아니라, 사용자가 앱 화면을 열어 둔 세션에서만 실행됩니다. 입력 작업 중이거나 직전 갱신 후 60초가 지나지 않은 경우에는 API 과호출을 막기 위해 건너뜁니다.

PC 화면의 **가격 조회 옵션**은 자동갱신 상태와 현재가 갱신 방식 안내를 보여줍니다. 자동갱신을 켜고 끄는 실제 스위치는 화면 상단에만 있습니다.

## 자동 저장 방식

공개 배포 버전은 로그인한 계정마다 `main` 포트폴리오 하나를 자동으로 사용합니다.

사용자가 보유 종목, 거래내역, 현금, 환율, 가격 정보를 변경하면 앱은 해당 계정의 포트폴리오를 자동 저장합니다. 저장이 성공하면 화면에 **저장됨** 상태가 표시됩니다. 저장에 실패해도 화면 입력값은 유지되며, 다음 변경 또는 새로고침 때 다시 저장을 시도할 수 있습니다.

공개 앱에서는 사용자가 다른 사람의 포트폴리오를 선택하거나 불러오는 UI를 제공하지 않습니다. 데이터는 Supabase Auth 사용자 ID 기준으로 분리됩니다.

## 보안과 개인정보

앱은 사용자별 데이터 분리를 위해 Supabase Auth와 Row Level Security 정책을 사용합니다.

운영 원칙은 다음과 같습니다.

- 사용자는 본인 계정으로 로그인한 뒤 본인 포트폴리오만 사용합니다.
- 공개 앱에는 Supabase service role key를 사용하지 않습니다.
- API key, 비밀번호, Supabase secret key는 GitHub 코드에 저장하지 않습니다.
- 일반 사용자는 Streamlit Cloud 관리 권한을 갖지 않습니다.
- 운영자 계정으로 접속한 브라우저에서는 Streamlit Cloud의 **Manage app** 메뉴가 보일 수 있습니다.

## 자주 묻는 질문

### 회원가입 후 바로 로그인되지 않습니다.

Supabase Auth 설정에서 이메일 확인이 켜져 있으면 인증 메일을 확인한 뒤 로그인해야 합니다. 메일함과 스팸함을 확인하세요.

### 가격 갱신을 눌렀는데 일부 종목이 실패합니다.

무료 데이터 소스 제한, 일시적인 네트워크 실패, 종목 코드 누락 때문에 발생할 수 있습니다. 잠시 후 다시 시도하거나 종목 티커가 올바른지 확인하세요. 실패한 종목은 마지막 정상 가격이 있으면 그 가격을 유지합니다.

### 오늘 변동이 0 또는 비어 보입니다.

오늘 변동은 최근 제공 가격과 전일 종가가 모두 있어야 계산됩니다. 데이터 소스가 전일 종가를 제공하지 않거나 최신 행만 반환하면 일간 변동이 0 또는 미표시로 보일 수 있습니다.

### 실제 기록과 과거 재구성이 다르게 보입니다.

실제 기록은 앱에 저장된 실제 스냅샷입니다. 과거 재구성은 사용자가 입력한 과거 보유현황과 무료 가격 데이터를 기준으로 다시 계산한 값입니다. 두 기능은 목적이 다르므로 결과가 다를 수 있습니다.

### 앱 하단 또는 브라우저에 Manage app이 보입니다.

Streamlit Cloud 운영 권한이 있는 계정으로 접속하면 관리 메뉴가 보일 수 있습니다. 일반 사용자에게 Streamlit workspace 권한을 주지 않으면 일반 이용자 화면에서는 관리 메뉴가 노출되지 않습니다.

## 한계와 유의사항

- 이 앱은 투자 조언, 매수/매도 추천, 세무 자문을 제공하지 않습니다.
- 무료 가격 데이터는 실시간성과 정확성을 보장하지 않습니다.
- 수수료, 세금, 배당, 환전 비용, 입출금은 사용자가 입력한 거래와 현금 원장 기준으로 계산합니다.
- 세금 집계는 사용자가 입력한 세금 값의 합계이며 실제 세무 신고용 계산이 아닙니다.
- 과거 보유현황 재구성은 투자성과 수익률이 아니라 입력 스냅샷 기준 평가액 변화입니다.
- 장기간 다종목 과거 재구성은 데이터 조회량이 많아 시간이 오래 걸릴 수 있습니다.

## 운영자 배포 메모

공개 사용자용 Streamlit Cloud 앱 설정은 다음을 기준으로 합니다.

```text
Main file path: app/public_portfolio_dashboard.py
Python version: 3.12
```

Streamlit Cloud Secrets에는 아래 값을 사용합니다.

```toml
SUPABASE_URL = "https://your-project.supabase.co"
SUPABASE_PUBLISHABLE_KEY = "your-publishable-key"
PUBLIC_USER_AUTH = true
AUTH_SESSION_SECRET = "32자 이상의 임의 문자열"
```

`SUPABASE_PUBLISHABLE_KEY` 대신 `SUPABASE_ANON_KEY`를 사용할 수 있습니다. 공개 앱에는 `SUPABASE_SERVICE_ROLE_KEY`를 넣지 않습니다.

`AUTH_SESSION_SECRET`은 로그인 유지 쿠키를 암호화하는 앱 전용 비밀값입니다. Supabase key나 앱 비밀번호를 재사용하지 말고 32자 이상으로 새로 만드세요. 이 값이 없으면 로그인 유지 체크박스는 비활성화됩니다.

한국투자증권 Open API 시세 조회를 사용하려면 Streamlit Cloud Secrets에 아래 값을 추가합니다. 실전투자 앱키를 발급받은 경우 `KIS_ENV = "real"`로 둡니다.

```toml
KIS_API_ENABLED = true
KIS_ENV = "real"
KIS_APP_KEY = "한국투자증권 Open API AppKey"
KIS_APP_SECRET = "한국투자증권 Open API AppSecret"
```

KIS 조회가 실패하거나 특정 해외 종목 거래소 매핑이 맞지 않으면 앱은 기존 yfinance 또는 FinanceDataReader 조회로 자동 fallback합니다.

배포 전 Supabase SQL Editor에서 `docs/supabase_migration_v2_auth_rls.sql`의 SQL 전체 내용을 복사해 실행합니다. SQL Editor에는 파일 경로가 아니라 SQL 내용을 붙여넣어야 합니다.

Secrets 또는 의존성을 바꾼 뒤에는 Streamlit Cloud에서 Reboot 또는 Redeploy를 실행해야 새 설정이 반영됩니다.
